💡 Key question — predict the error
A missing file makes .NET throw FileNotFoundException ("can't find it"). But the real crash was BadImageFormatException ("found it — but it isn't a loadable image"). The DLL isn't missing; it's right there, bundled inside the 118 MB exe. So what is special about loading a native DLL from inside another file?
⚠️ The rule that bites you
A native / mixed-mode DLL cannot be loaded from a single-file bundle. It must exist as a real file on disk so the Windows loader can open it by path. This is a documented .NET single-file limitation — and exactly what SPRecorder's README warns about.
📐 Takeaways & trade-offs
  • ✅ Fix: dotnet publish -c Release -r win-x64 --self-contained true (no single-file), then ship the whole folder.
  • ⚖️ Trade-off: multi-file is ~255 files — they must travel together, and ScreenRecorderLib.dll must stay beside the exe.
  • 📦 For non-coders: wrap that folder in a portable ZIP with a launcher so they just double-click.
  • 🚫 Never tick "Produce single file" in Rider while ScreenRecorderLib is a dependency.
